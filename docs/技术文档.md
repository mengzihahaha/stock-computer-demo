# 股票因子计算器 —— 技术文档

## 1. 项目概述

一个基于 Streamlit 的 Web 应用：输入股票代码，自动获取历史行情数据，计算常用量化因子（均线、动量、波动率、RSI），并以交互式图表展示，支持结果导出为 CSV。支持**美股**（yfinance）和 **A股**（akshare）两个市场。

- **定位**：个人学习 + 求职作品项目
- **核心能力**：数据获取 → 因子计算 → 可视化展示 → 数据导出

---

## 2. 技术栈

| 层次 | 技术 | 作用 |
|------|------|------|
| Web 界面 | Streamlit 1.40 | 把 Python 脚本变成交互式网页 |
| 数据处理 | pandas 2.2 / numpy 2.1 | 因子计算核心 |
| 美股数据源 | yfinance 0.2.50 | 获取美股行情 |
| A股数据源 | akshare 1.16.98 | 获取 A股行情（东方财富 + 新浪财经）|
| 运行环境 | Python 3.12 | —— |

---

## 3. 系统架构

采用**分层 + 关注点分离**设计，三个模块各司其职，互不耦合：

```
┌─────────────────────────────────────────┐
│                app.py                     │  界面层：只管 UI 和串联
│         (Streamlit 界面 + 入口)            │
└───────────────┬───────────────┬──────────┘
                │               │
                ▼               ▼
        ┌──────────────┐ ┌──────────────┐
        │   data.py    │ │  factors.py  │
        │  数据获取层   │ │  因子计算层   │
        └──────┬───────┘ └──────────────┘
               │            (纯函数，无副作用)
               ▼
        ┌──────────────────────┐
        │  yfinance / akshare  │  外部数据源
        └──────────────────────┘
```

**设计价值（面试可讲）**：
- 换数据源只改 `data.py`，界面和计算不受影响
- `factors.py` 是纯函数，极易写单元测试
- 各层职责单一，代码可读、可维护、可扩展

## 4. 模块详解

### 4.1 data.py —— 数据获取层

**职责**：从外部数据源获取行情，整理成统一格式的 DataFrame。

**统一输出格式**：索引为日期，列为 `open, high, low, close, volume`。

**核心函数**：

| 函数 | 说明 |
|------|------|
| `get_stock(market, symbol, period)` | 统一入口，按市场分发到具体实现 |
| `get_us_stock(symbol, period)` | 美股，走 yfinance |
| `get_cn_stock(symbol, period)` | A股，双数据源容错 |
| `_retry(func, times, wait)` | 通用重试工具，应对网络抖动 |
| `_to_sina_symbol(symbol)` | 6位代码转新浪格式（sh/sz/bj 前缀）|

**健壮性设计（亮点）**：
- **自动重试**：网络请求失败自动重试若干次
- **双数据源容错**：A股优先东方财富，失败自动降级到新浪财经，单源被限流时仍可用

### 4.2 factors.py —— 因子计算层

**职责**：输入行情，输出因子。所有函数都是**纯函数**（输入相同则输出相同，不联网、不改全局状态），因此极易测试。

| 因子函数 | 金融含义 | 计算方法 |
|----------|----------|----------|
| `moving_average(close, 20)` | 趋势方向 | 20日收盘价滚动均值 |
| `momentum(close, 20)` | 涨跌力度 | 20日累计收益率（%）|
| `volatility(close, 20)` | 风险大小 | 日收益率的滚动标准差 |
| `rsi(close, 14)` | 超买/超卖 | 14日相对强弱指标 (0~100) |
| `compute_all_factors(df)` | —— | 组合以上，一次算全部因子 |

### 4.3 app.py —— 界面层

**职责**：只负责 UI 渲染和串联各模块，不含业务逻辑。

**关键点**：
- `@st.cache_data(ttl=3600)`：缓存数据 1 小时，避免重复请求
- 侧边栏输入参数（市场、代码、时间范围）
- 主区域用 `st.line_chart` / `st.dataframe` 展示图表和数据
- `try/except` 捕获异常，友好提示而非崩溃
- `st.download_button` 支持结果导出 CSV（UTF-8-SIG 防中文乱码）

## 5. 数据流

```
用户输入(市场/代码/周期)
   → app.py 调用 load_data() [带缓存]
   → data.get_stock() 分发
   → 具体数据源获取 + 重试/降级
   → 返回统一格式 DataFrame
   → factors.compute_all_factors() 计算因子
   → app.py 渲染图表 + 表格 + 下载按钮
```

## 6. 运行方式

### 安装依赖
```bash
python3 -m venv venv
source venv/bin/activate        # Windows CMD: venv\Scripts\activate
pip install -r requirements.txt
```

### 启动应用
```bash
streamlit run app.py
```
浏览器访问 `http://localhost:8501`。

### 单独测试模块
```bash
python factors.py   # 测试因子计算（不联网）
python data.py      # 测试数据获取（联网）
```

## 7. 已知问题与排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 美股报错 / HTTP 429 | Yahoo 对共享 IP 限流 | 等待 10-30 分钟后重试 |
| A股 RemoteDisconnected | 东方财富限流 | 已加新浪财经自动降级；重试即可 |
| WSL 下 IPv6 连接失败 | WSL 的 IPv6 常不通 | 数据源已通过重试/降级兜底 |
| `venv\Scripts\activate` 无效 | 在 WSL/Linux 用了 Windows 命令 | 改用 `source venv/bin/activate` |
| pip 报 externally-managed | 未激活 venv，装到系统 | 先激活虚拟环境 |
| `/mnt/c` 装依赖极慢 | WSL 访问 Windows 盘 IO 慢 | venv 建在 Linux 主目录（如 `~`）|
